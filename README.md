## Телеграм‑бот анкета по стоматологии (Docker)

Бот проводит короткую анкету (25 вопросов по клику), автоматически считает баллы, определяет статус риска стоматологических проблем и выдаёт индивидуальные рекомендации. В конце пользователь может отправить номер телефона, который пересылается администратору.

Логика оценки:
- **0–10 баллов** — лёгкие нарушения.
- **11–20 баллов** — умеренные нарушения.
- **21+ баллов** — тяжёлые нарушения.

Анкета построена по 6 упрощённым категориям:
- **Общие данные ребёнка**
- **Питание и сладости**
- **Привычки перед сном**
- **Гигиена полости рта**
- **Полезные продукты**
- **Состояние зубов и отношение к гигиене**

Все вопросы реализованы как **варианты по кнопкам**, чтобы пройти анкету можно было за 2–3 минуты без ввода текста.

### Подготовка

1. Установите Docker и Docker Compose (при необходимости).
2. В корне проекта создайте файл `.env` на основе `.env.example`:

```env
BOT_TOKEN=ВАШ_TELEGRAM_BOT_TOKEN
ADMIN_CHAT_ID=123456789
```

`ADMIN_CHAT_ID` — это числовой id чата администратора (можно взять из логов бота или через @userinfobot).

### Сборка Docker‑образа

```powershell
cd c:\Users\Ф\Desktop\freelance\stomatest
docker build -t stomatest-bot .
```

### Запуск контейнера

```powershell
docker run --rm `
  --env-file .env `
  --name stomatest-bot `
  stomatest-bot
```

Бот запустится и подключится к Telegram. Остановить можно по `Ctrl+C` (если запущен в foreground) или `docker stop stomatest-bot` (если запущен в фоне).

### Как работает бот

- Пользователь отправляет команду **/start**.
- Бот показывает **дисклеймер** (отказ от ответственности) и кнопку «Согласен, начать».
- Далее идёт **анкета из 25 вопросов** с вариантами ответов по кнопкам.
- За каждый ответ начисляется от 0 до 3 баллов, формируется общий итог.
- По итоговой сумме бот определяет **статус** и выдаёт краткий **индивидуальный отчёт с рекомендациями**:
  - при лёгких/умеренных нарушениях — рекомендации по гигиене и питанию;
  - при тяжёлых нарушениях — акцент на необходимости очной консультации стоматолога.
- После отчёта бот предлагает **оставить номер телефона**:
  - специальная кнопка запрашивает контакт, телефон отправляется администратору (по `ADMIN_CHAT_ID`);
  - можно отказаться кнопкой «Не оставлять телефон».

Файл `bot.py` содержит всю основную логику анкеты, подсчёта баллов, формирования статуса и отправки телефона админу.
